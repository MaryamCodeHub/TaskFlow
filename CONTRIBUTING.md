# Contribution Guidelines

Thank you for your interest in contributing to TaskFlow!

## How to Contribute
1. Fork the repository and clone it locally.
2. Create an issue to discuss your proposed change or assign yourself to an existing issue.
3. Create a branch off `development` using the naming convention `feature/<name>` or `bugfix/<name>`.
4. Commit your changes with meaningful commit messages.
5. Push the branch and create a Pull Request against the `development` branch.
6. Address any feedback during code review.
7. Once approved, a maintainer will merge your PR.

Ensure your code follows the structured patterns in `src/` and that any new features are covered by tests in `tests/`.
