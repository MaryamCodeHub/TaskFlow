# TaskFlow

TaskFlow is a basic task management application designed to help users organize their daily activities effectively.

## Repository Structure
- `src/`: Core source code and feature implementation.
- `docs/`: Project documentation and architecture notes.
- `tests/`: Automated unit and integration tests.

## Development Workflow
This project utilizes a feature-branching Git workflow:
1. `main`: Contains stable, production-ready code.
2. `development`: Used as an integration branch for ongoing work.
3. Feature branches (`feature/*`) and Bugfix branches (`bugfix/*`) are branched off `development` and merged back via Pull Requests.

## Contribution Guidelines
1. Fork the repository and clone it locally.
2. Create a new branch from `development` for your feature or bug fix.
3. Ensure commit messages are clear, descriptive, and follow conventional commit formats.
4. Push your branch and open a Pull Request against the `development` branch.
5. Request a code review before merging.


## Authentication
Password handling is securely managed.
